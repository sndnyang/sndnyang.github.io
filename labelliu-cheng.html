firefly client

客户端新建项目

  1. 拷贝 模板文件夹 副本， 改名为 项目名-项目编号
  2. 将此文件夹放到 firefly 控制的主目录下，
    示例:
    下载分支 LB03_直付通项目， 在该分支中从模板文件夹建立新项目。
  3. 右键添加到firefly控制中。

firefly server explorer

建立link链接

  1. 选中项目（一般选主分支下的XXX_YYY项目 这个分支）
  2. 点创建链接，选择分支（选择相应REL分支） 例如 LB_03直付通系统——MainBranch——LB03_直付通_REL——LB03_直付通_DEV 或 LB03_直付通MIDRUN_DEV 如图：
  3. 选择源分支及目标分支的路径
    源分支：选择到产品下一级的项目，如PAYFEE-EPAY
    目标分支：选择到项目-程序（文档）-XX 一级。
    如图：
  4. 点击中间的 >> 符号，即可链接过去。选中 Readonly后，下一步。
  5. 后续填写，完成。

开发环境

项目编辑常用命令

  1. hff create 文件名/文件夹名/.(指当前全部)
  2. hff edit/unedit 文件名/文件夹名/.(指当前全部)
  3. hff submit 文件名/文件夹名/.(指当前全部)
  4. hff list-out 目录，查看是否有edit，但没有submit的文件
  5. hff list-hijacked 目录，查看是否有强行非法修改的文件
  6. hff list-nwf 目录，查看是否有没有添加到Firefly管理的文件
  7. hff diffs 文件名，查看版本区别

开发环境建label

  1. 编辑文件列表
  2. 运行脚本

编辑文件列表

  1. devfile.txt
    要提交文件的相对路径，模板见 label工具/devfile.txt 格式: 
    项目名-项目编号\path\文件

    path由建立link链接时的路径加代码文件所在路径组合。
    示例： 

    否认交易处理系统维护改进-T1317341\程序\服务器程序\EPAY\script\EPAY_CHL_90210_Script.cfg

    EPAY之前是目标分支的路径，从EPAY到之后是源分支中链接路径。

运行脚本

  1. makelabel_dev.bat
    修改

  2. PRJ_NAME 项目

  3. REL_BRANCH 编译分支名

  4. LABEL_NAME 标签名

  5. 执行

编译环境

下载

  1. 登陆 99.1.76.245 midsrv/midsrv
    cd tmp
  2. downlaod_lbl.sh label名
  3. 覆盖相应项目的流程文件 或是 添加
  4. hff create 文件名/文件夹名/.(指当前全部)
  5. hff edit 文件名/文件夹名/.(指当前全部)
  6. hff submit 文件名/文件夹名/.(指当前全部)

编译环境打label

  1. relfile.txt
    要提交文件的相对路径，模板见 label工具/devfile.txt
    格式:
    path\文件 path为文件完整路径
    示例： 

    midsrv\node\PAYFEE\EPAY\script\EPAY_CHL_90210_Script.cfg

  2. makelable_rel.bat 修改

  3. PRJ_NAME 项目

  4. REL_BRANCH 编译分支名

  5. LABEL_NAME 标签名

  6. 执行

运行实例label

  1. 进入到对应的项目下(如LB01总行中间业务->LB01通信前置网关)
  2. 右键,如右键LB01_通信前置网关_项目,选择Label
  3. 填写好Name,如T1210101_LB01(BLGW)_V01.00_L01(最好选择下面的Inherit...)
  4. 之后的一步选择所更新的文件
