ksh 判断

一. 基础比较

  1. 运算符号代表意义
  2. = 等于 应用于：整型或字符串比较 如果在[] 中，只能是字符串
  3. != 不等于 应用于：整型或字符串比较 如果在[] 中，只能是字符串
  4. < 小于 应用于：整型比较 在[] 中，不能使用 表示字符串
  5. > 大于应用于：整型比较 在[] 中，不能使用 表示字符串
  6. -eq 等于 应用于：整型比较
  7. -ne 不等于 应用于：整型比较
  8. -lt 小于 应用于：整型比较
  9. -gt 大于 应用于：整型比较
  10. -le 小于或等于 应用于：整型比较
  11. -ge 大于或等于 应用于：整型比较
  12. -a 双方都成立（and） 逻辑表达式 –a 逻辑表达式
  13. -o 单方成立（or） 逻辑表达式 –o 逻辑表达式
  14. -z 空字符串
  15. -n 非空字符串
  16. 逻辑的『和(and)』『或(or)』
  17. && 逻辑的 AND 的意思
  18. || 逻辑的 OR 的意思

二. 特殊比较

  1. 关于档案与目录的侦测逻辑卷标！
  2. -f 常用！侦测『档案』是否存在 eg: if [ -f filename ]
  3. -d 常用！侦测『目录』是否存在
  4. -b 侦测是否为一个『 block 档案』
  5. -c 侦测是否为一个『 character 档案』
  6. -S 侦测是否为一个『 socket 标签档案』
  7. -L 侦测是否为一个『 symbolic link 的档案』
  8. -e 侦测『某个东西』是否存在！
  9. 关于程序的逻辑卷标！
  10. -G 侦测是否由 GID 所执行的程序所拥有
  11. -O 侦测是否由 UID 所执行的程序所拥有
  12. -p 侦测是否为程序间传送信息的 name pipe 或是 FIFO （老实说，这个不太懂！）
  13. 关于档案的属性侦测！
  14. -r 侦测是否为可读的属性
  15. -w 侦测是否为可以写入的属性
  16. -x 侦测是否为可执行的属性
  17. -s 侦测是否为『非空白档案』
  18. -u 侦测是否具有『 SUID 』的属性
  19. -g 侦测是否具有『 SGID 』的属性
  20. -k 侦测是否具有『 sticky bit 』的属性
  21. 两个档案之间的判断与比较；例如[ test file1 -nt file2 ]
  22. -nt 第一个档案比第二个档案新
  23. -ot 第一个档案比第二个档案旧
  24. -ef 第一个档案与第二个档案为同一个档案（ link 之类的档案）
